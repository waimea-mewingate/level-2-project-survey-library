# Project Requirements

## Identified Problem or Need

My project will attempt to solve the problem of keeping track of survey instruments / avoiding job overlaps that require the use of said instrumants.

## End-User Requirements

My end-users are my dad and his workmates at Newton Survey. I talked to my dad, and they will need the program to be easy to use, clear (check statuses at a glance), and simple. This can make navigating between functions (such as instrument availability) as smooth and the system as usable as possible.

## Proposed Solution

I intend to create a system that allows the end-users to schedule the use of and check the status of survey instruments. It should be as simple as possible. I will keep this system targeted at their workplace specifically as it will be using data from instruments of a combination that maybe only their workplace has. 


# Relevant Implications

## Usability

How easy to navigate and use the system is to anybody while minimising confusion and frustration.

### Relevance to the System

This is relevant as systems that are confusing or hard to navigate will reduce the ability for users to use the system and may mean that this project will have no value (doesn't perform).

### Impact / Considerations

To make the site easy to use, I will need to consider layout and system navigation. This will affect the project dramatically depending on how it is approached. A way to do this would be to keep things to a few pages- most functions accessible either on the home page or through a single button. I will also try to add feedback where possible. 


## Accessibility

How useable the site is for people regardless of their abilities or device type.

### Relevance to the System

This is relevant as my users will likely be using a variety of devices like their desktop PCs and their phones, and the workplace also contains multiple people that are colour-blind, including my dad!

### Impact / Considerations

To increase the accesssibility of my site, I will need to consider my colour scheme- keeping it minimal (black/white with a single highlight colour?) and contrasting dark with light to keep things clear for the colour-blind users. I will also try to keep my site sectioned using semantic HTML tags to allow for device transition and any shuffling that might come with varied screen sizes. 


## End User

How much the system is built with those that will use it in mind- their needs. 

### Relevance to the System

This is relevant because the system must function for its intended uses while still being easy to use. Otherwise, I have made a project that does not fill the intended requirement and (likely) will not end up being utilised.

### Impact / Considerations

I plan to meet the end-user needs by getting feedback as much as I can- making sure I stay on track. From this, I can glean information on what functions need priority, and can act accordingly. I will also attempt to make my site as simple as I can, as this will help to speed up the usage of the site (and help to meet the useability heuristics!)


## Functionality

Creating a system that works.

### Relevance to the System

Systems that don't work are useless! Systems that are slow or broken are frustrating and hinder the success of the project.

### Impact / Considerations

To adhere to the heuristic of functionality, I will need to test every function (buttons/forms) as they are added and improved upon to make sure that they work as they should, as well as test the system on mobile and desktop.


## Aesthetics

The visual design of the system.

### Relevance to the System

Having an aesthetic suited to the task is important for user experience, and consistency with styling is needed to avoid confusion.

### Impact / Considerations

To improve aesthetics, I will need to keep in mind minimalism and consistency when designing my site- a limited palette and simple, clean layout as much as possible while keeping everything mostly the same across pages. 
This will also be needed to be considered in tandem with accessibility- in terms of colour palette, I believe that using black/white/grey with an accent colour (yellow? Most colour-blind people can see yellow) will help to keep things clear for my colourblind users- I will need to be thinking more about light and dark than colour.
