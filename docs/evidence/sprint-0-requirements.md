# Project Requirements

## Identified Problem or Need

My project will attempt to solve the problem of keeping track of survey instruments / avoiding job overlaps that require the use of said instrumants.


## End-User Requirements

My end-users are my dad and his workmates at Newton Survey, they will need it to be easy to use, clear (check statuses at a glance). It will need to be as fast as possible to check what they need to check (availability etc.)


## Proposed Solution


I intend to create a system that allows the end-users to schedule the use of and check the status of survey instruments. It should be as simple as possible. I will keep this system targeted at their workplace specifically as it will be using data from instruments of a combination that maybe only their workplace has. 


# Relevant Implications

## Usability

How easy to navigate and use the system is to anybody while minimising confusion and frustration.

### Relevance to the System

This is relevant as this system is meant to show things at a glance, and if a system is frustrating it will very much reduce effectiveness.

### Impact / Considerations

I will need to consider layout and system navigation. This will affect the project dramatically depending on how it is approached. A way to do this would be to keep things to a few pages- only two or three. 



## Accessibility

How useable the site is for people regardless of their abilities or device type.

### Relevance to the System

This is relevant as my users will be using a variety of devices, and the workplace also contains many people that are colour-blind!

### Impact / Considerations

I will need to consider my colour scheme- keeping it minimal and contrasting dark with light to keep things clear. 
I will also try to keep my site sectioned (semantic HTML tags) to allow for device transition and any shuffling that might come with varied screen sizes. 


## End User

How much the system is built with those that will use it in mind- their needs. 

### Relevance to the System

This is relevant because the system must function for its intended uses while still being easy to use by my users.

### Impact / Considerations

I will do this by prioritising tasks- main ones first. I also will try to display everything in simplified form to be as fast to use as possible- status check being displayed as an icon etc, booking accessible from a button. Keeping everything in one place. 



## Functionality

Creating a system that works.

### Relevance to the System

Systems that don't work are useless! Systems that are slow or broken are frustrating and hinder the success of the project.

### Impact / Considerations

I will need to test every function as it is added and improved upon to make sure that it works as it should, as well as heving to test the system on mobile and desktop.



## Aesthetics

The visual design of the system.

### Relevance to the System

A consistent colour palette and style matter to help avoid confusion.

### Impact / Considerations

I will need to keep in mind minimalism and consistency when designing my site- a limited palette and simple, clean layout as much as possible.
